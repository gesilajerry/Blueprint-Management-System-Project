## 一、项目概述

### 1.1 项目背景

公司当前图纸管理存在以下核心问题：

| 问题 | 影响 |
|------|------|
| 全部图纸加密外发 | 供应商无法加工，效率下降；部分供应商拒绝接单 |
| 全部图纸不加密 | 核心图纸泄露风险，保密管控形同虚设 |
| 无版本追溯机制 | 版本更替原因、变更内容不可查 |
| 无知识沉淀 | 图纸背景、用途、技术参数未结构化记录 |
| 无分类分级 | "要么全管死、要么全放敞" |
| 项目协作不清晰 | 工程师不知道项目内谁负责什么 |
| 工作量难以量化 | 员工贡献无法量化考核 |

### 1.2 建设目标

> 建立**分类分级**的图纸全生命周期管理体系，实现**保密有级、版本可溯**，先把图纸管起来。

> 外发环节的加密/保密协议等操作在现有系统/流程中处理，不在本系统职责范围内。

### 1.3 设计原则

- **精简优先**：内部自研，轻量化部署，不过度设计
- **分类分级**：按部件重要性差异化保密策略，避免一刀切
- **分期交付**：核心功能优先一期上线，逐步完善
- **实用导向**：减少设计师额外操作负担，尽量自动提取/建议
- **动态权限**：权限可配置，实时生效，满足灵活管理需求

## 二、保密分类体系

### 2.1 图纸保密等级

| 等级 | 名称 | 定义 |
|------|------|------|
| **A** | 核心机密 | 涉及核心技术、核心部件、核心功能的图纸 |
| **B** | 重要 | 重要功能部件，非核心技术但对产品有重要影响 |
| **C** | 一般 | 标准件、通用件、零散小件、非核心部件 |

> 外发时的加密、保密协议等合规操作由外部门/系统处理，本系统专注于图纸本身的分类定级与版本管理。

### 2.2 保密等级判定规则

**判定方式：两级审核流程**

```
上传人初步判断 → 系统辅助建议 → 审定人最终定级 → 记录调整日志
```

**第一步：上传人初步判断（必填）**
- 上传人必须填写"是否涉及核心部件"（是/否）
- 并给出初定保密等级建议（A/B/C）
- 系统根据核心部件词库匹配，自动提示建议等级（仅供参考）

**第二步：审定人最终定级（必审）**
- 审定人审核上传人的初定结果
- 可维持原级别，也可调整
- 调整时必须填写调整原因（记录日志）
- 审定完成后，保密等级正式生效

### 2.3 保密等级变更

图纸正式定级后，如需调整保密等级：
- 等级上调（A→B / A→C / B→C）：需填写变更原因 + 审批确认
- 等级下调（C→B / B→A / C→A）：需登记变更原因，不强制审批
- 所有变更历史全程可查

## 五、技术架构

### 5.1 技术栈

| 层次 | 推荐方案 | 说明 |
|------|---------|------|
| 前端 | Vue3 + Element Plus + Pinia | 组件化开发，状态管理 |
| 后端 | Python FastAPI + SQLAlchemy | 异步高性能，开发效率高 |
| 数据库 | SQLite (开发) / PostgreSQL (生产) | 结构化数据，稳定 |
| 文件存储 | 服务器本地磁盘 | 简单直接，500MB内文件无压力 |
| 认证 | JWT + 动态 RBAC | 内部系统，不过度复杂 |
| 部署 | Docker + Docker Compose | 一键部署，维护简单 |

### 5.2 系统架构图

```
┌─────────────┐
│   浏览器     │  ← Vue3 前端 (Port 3000)
└──────┬──────┘
       │ HTTP/REST
┌──────▼──────┐
│  FastAPI    │  ← 后端服务 (Port 8000)
│  后端       │
└──────┬──────┘
       │
┌──────▼──────┐     ┌──────────────┐
│ SQLite/PG   │     │  文件存储     │
│   数据库     │     │  /data/drawings │
└─────────────┘     └──────────────┘
```

## 七、风险与注意事项

| 风险 | 应对 |
|------|------|
| 设计师抗拒操作负担 | 尽量减少必填字段，系统自动填充能省的环节就省 |
| 图纸文件大，上传慢 | 文件存储直接走磁盘；前端大文件分片上传 |
| 历史图纸导入量大 | 初期手动导入为主，不做自动迁移 |
| 编号规则与现有习惯冲突 | 可按需调整编号格式配置 |

## 八、术语表

| 术语 | 定义 |
|------|------|
| 图纸保密等级 | A/B/C 三级，分别对应核心机密/重要/一般 |
| 核心部件 | 对产品功能、核心竞争力有重要影响的关键部件 |
| 外发 | 将图纸提供给公司外部单位（供应商/客户） |
| 版本更替 | 图纸的迭代更新，保留历史版本 |
| 项目组 | 按产品/项目划分的团队，负责人可查看组内所有图纸 |
| 工作量统计 | 统计用户的创建图纸数、上传版本数、审核数 |
