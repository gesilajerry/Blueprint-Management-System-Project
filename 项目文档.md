# 图纸管理系统 - 项目文档

**项目名称：** 图纸管理系统 (Blueprint Management System)
**版本：** V1.0
**状态：** 功能开发完成，测试优化阶段
**更新日期：** 2026-04-11

---

## 一、建设目标

建立**分类分级**的图纸全生命周期管理体系，实现**保密有级、版本可溯**。

### 核心问题解决

| 问题 | 解决方案 |
|------|----------|
| 全部图纸加密外发 | 分类分级，差异化保密策略 |
| 无版本追溯机制 | 完整版本历史记录 |
| 无知识沉淀 | 技术参数结构化记录 |
| 无分类分级 | A/B/C 三级保密体系 |
| 项目协作不清晰 | 项目组管理，负责人权限隔离 |
| 工作量难以量化 | 用户贡献统计（创建/上传/审核数） |
| 权限调整不灵活 | 19个权限键动态配置，实时生效 |

## 二、技术架构

### 2.1 技术栈

| 层次 | 技术选型 | 说明 |
|------|---------|------|
| **前端** | Vue3 + Element Plus + Pinia + Vite | 组件化开发，状态管理 |
| **后端** | Python FastAPI + SQLAlchemy | 异步高性能，开发效率高 |
| **数据库** | SQLite (开发) / PostgreSQL (生产) | 结构化数据，稳定可靠 |
| **文件存储** | 本地磁盘 | 支持大文件上传（500MB） |
| **认证** | JWT Token + 动态 RBAC | 内部系统，高效安全 |

### 2.2 系统架构

```
┌─────────────┐
│   浏览器     │  ← Vue3 前端 (Port 3000)
└──────┬──────┘
       │ HTTP/REST API
┌──────▼──────┐
│  FastAPI    │  ← 后端服务 (Port 8000)
│  后端       │
└──────┬──────┘
       │
┌──────▼──────┐     ┌──────────────┐
│ SQLite/PG   │     │  文件存储     │
│   数据库     │     │  /data/drawings │
└─────────────┘     └──────────────┘
```

## 四、保密分类体系

### 4.1 保密等级

| 等级 | 名称 | 定义 |
|------|------|------|
| **A** | 核心机密 | 涉及核心技术、核心部件、核心功能的图纸 |
| **B** | 重要 | 重要功能部件，非核心技术但对产品有重要影响 |
| **C** | 一般 | 标准件、通用件、零散小件、非核心部件 |

### 4.2 两级审核流程

```
上传人初步判断 → 系统辅助建议 → 审定人最终定级 → 记录调整日志
```

## 五、数据模型

### 5.1 核心实体（11张表）

| 表名 | 说明 |
|------|------|
| `users` | 用户账户 |
| `roles` | 角色定义（含权限JSON） |
| `departments` | 部门字典 |
| `products` | 产品/项目 |
| `project_groups` | 项目组 |
| `project_group_members` | 项目组成员 |
| `drawings` | 图纸元数据 |
| `drawing_versions` | 图纸版本 |
| `drawing_relations` | 图纸关联（上下游） |
| `drawing_core_parts` | 图纸核心部件 |
| `core_part_keywords` | 核心部件关键词 |
| `system_logs` | 系统日志 |
| `review_history` | 审核历史 |

### 5.2 核心表结构

#### 用户表 `users`
```sql
id                    UUID PRIMARY KEY
username              VARCHAR(50) UNIQUE  -- 用户名
name                  VARCHAR(50)         -- 姓名
password_hash         VARCHAR(255)        -- 密码哈希
department_id         UUID                -- 所属部门
role_id               UUID                -- 角色
email                 VARCHAR(100)        -- 邮箱
phone                 VARCHAR(20)         -- 手机
status                ENUM('active','disabled')
created_at            TIMESTAMP
last_login_at         TIMESTAMP
```

#### 角色表 `roles`
```sql
id                    UUID PRIMARY KEY
name                  VARCHAR(50)         -- 角色名称
code                  VARCHAR(50) UNIQUE  -- 角色编码
permissions           JSONB               -- 19个权限键值对
status                ENUM('active','disabled')
created_at            TIMESTAMP
```

#### 图纸表 `drawings`
```sql
id                    UUID PRIMARY KEY
drawing_no            VARCHAR(50) UNIQUE  -- 图号
name                  VARCHAR(200)        -- 图纸名称
product_id            UUID                -- 所属产品
department_id         UUID                -- 所属部门
confidentiality_level ENUM('A','B','C')   -- 保密等级
is_core_part          BOOLEAN             -- 是否核心部件
created_by            UUID                -- 创建人
created_at            TIMESTAMP
updated_at            TIMESTAMP
status                ENUM('active','archived','deleted')
review_status         ENUM('pending','approved','rejected')
```

#### 项目组表 `project_groups`
```sql
id                    UUID PRIMARY KEY
name                  VARCHAR(100)        -- 组名
product_id            UUID                -- 关联产品
leader_id             UUID                -- 负责人
status                ENUM('active','archived')
created_at            TIMESTAMP
```

## 六、权限设计

### 6.1 角色划分（8种）

| 角色编码 | 角色名称 | 权限说明 |
|----------|----------|----------|
| `admin` | 管理员 | 全部功能，系统配置，用户管理 |
| `cto` | CTO | 查看所有图纸，保密审核，统计日志 |
| `project_manager` | 项目负责人 | 查看项目图纸，管理产品，工作量统计 |
| `engineer` | 工程师 | 创建/上传图纸，仅看自己图纸 |
| `designer` | 设计师 | 创建/上传图纸，仅看自己图纸 |
| `reviewer` | 审定人 | 保密等级审定，查看所有图纸 |
| `archive_manager` | 档案管理员 | 查看/下载所有图纸，查看日志 |
| `guest` | 访客 | 仅查看已授权图纸（无图纸权限） |

### 6.2 19个权限键

```
图纸查看: viewDashboard, viewDrawings, viewAllDrawings, viewProjectDrawings,
          viewOwnDrawings, viewPendingReview, viewAlevelDrawings
图纸操作: createDrawing, uploadVersion, downloadDrawing
保密审核: reviewConfidentiality
管理功能: manageUsers, manageRoles, manageProjectGroups, manageDepartments,
          manageProducts, manageCoreParts, viewLogs, workLog
```

### 6.3 动态权限配置

- 角色权限存储在数据库 `roles.permissions` JSON字段
- `RoleManage.vue` 可视化配置，实时生效
- 后端 `check_user_permission()` 优先读数据库，自定义权限为空时使用默认矩阵

## 九、API 设计

### 9.1 认证接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 用户登录 |
| POST | `/api/auth/logout` | 用户登出 |
| GET | `/api/auth/me` | 获取当前用户信息（含19个权限） |

### 9.2 图纸接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/drawings` | 图纸列表（分页、筛选、动态权限过滤） |
| POST | `/api/drawings` | 创建图纸 |
| GET | `/api/drawings/:id` | 图纸详情 |
| PUT | `/api/drawings/:id` | 更新图纸 |
| DELETE | `/api/drawings/:id` | 删除/作废图纸 |
| POST | `/api/drawings/:id/versions` | 上传新版本（multipart/form-data） |
| GET | `/api/drawings/:id/versions` | 版本历史 |
| GET | `/api/drawings/:id/download` | 下载图纸 |
| GET | `/api/drawings/export/csv` | 导出图纸数据为 CSV |

### 9.3 审核接口

| 方法 | 路径 | 说明 |
|------|------|------|

| POST | `/api/reviews/:id/reject` | 审核驳回 |
| GET | `/api/reviews/history/all` | 所有审核历史 |

### 9.4 项目组接口
